# @streetjs/http — HTTP server, routing, middleware pipeline

- **Package:** `@streetjs/http`
- **Consumers (StreetStudio):** `apps/api` (REST + webhook management endpoints)
- **Depends on:** `@streetjs/core`, `@streetjs/security` (error mapping, rate limit), `@streetjs/metrics`
- **Wave:** 3 (web & identity)

## Motivation

StreetStudio serves a REST API and must not implement its own HTTP server or
router. The web runtime is generic platform infrastructure.

## Required API surface

- `HttpModule` + server lifecycle (start/stop) wired to `@streetjs/core`.
- Routing: method + path params, typed handlers, route groups.
- Ordered middleware pipeline (auth, validation, rate limit, error mapping).
- Request/response abstractions; body parsing; typed input validation hook.
- Error-to-response mapping using the `@streetjs/security` taxonomy (non-disclosing).

## Acceptance criteria

- [ ] Routes dispatch by method/path with typed params; unmatched routes yield a standard 404.
- [ ] Middleware executes in declared order; a short-circuit (e.g. auth deny) halts the chain.
- [ ] Thrown taxonomy errors map to correct status codes and non-disclosing bodies.
- [ ] Request metrics/trace context are emitted via `@streetjs/metrics`.
- [ ] Server starts/stops cleanly under the container lifecycle.

## Non-goals

- No auth/session logic (compose `@streetjs/auth`); no WebSocket (see `@streetjs/realtime`).
