# @streetjs/rbac — role-based authorization (deny-by-default)

- **Package:** `@streetjs/rbac`
- **Consumers (StreetStudio):** every authorized action across REST and WebSocket
- **Depends on:** `@streetjs/core`
- **Wave:** 3 (web & identity)

## Motivation

StreetStudio enforces deny-by-default authorization scoped to an owning
organization before any action. The evaluation engine is generic; the specific
roles/policies are product configuration StreetStudio supplies.

## Required API surface

- Role/permission model: define roles, permissions, and scoped assignments.
- `authorize(subject, action, resource, scope)` → allow/deny, deny-by-default.
- Policy registration so products (StreetStudio) declare their actions/resources.
- Middleware/guard helper for `@streetjs/http` and realtime handlers.

## Acceptance criteria

- [ ] Unknown/unmatched action-resource pairs are denied by default.
- [ ] Decisions are evaluated within the given scope; a subject authorized in one scope is denied in another.
- [ ] Denials produce no side effects and a standard non-disclosing error.
- [ ] The same decision is reached regardless of surface (HTTP vs realtime) given identical inputs.
- [ ] Fully typed and deterministic; unit-testable without I/O.

## Non-goals

- No authentication (that is `@streetjs/auth`); no product-specific role catalog.
